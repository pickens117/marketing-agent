# AI Adoption Plan

Create a practical AI adoption plan for: $ARGUMENTS

Before answering:

- review `docs/company/company-context.md` if it exists
- use company-specific messaging, constraints, and approval requirements when available
- if company context is missing, say that the plan is generic and label assumptions clearly

Optimize for a marketing team that needs real operational guidance, not abstract strategy.

Use this structure:

1. Objective
2. Current-state assumptions
3. Best initial use cases
4. Recommended workflow changes
5. Roles and responsibilities
6. Prompting and QA guidance
7. Governance and risk controls
8. Enablement and training plan
9. Success metrics
10. 30-day rollout plan

Recommend a lightweight, low-risk starting point if the request is broad or early-stage.
